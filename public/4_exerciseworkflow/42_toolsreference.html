Tools Reference

Quick Reference for Workshop Tools

This page provides quick command syntax for all tools used in the 7-step workflow. Bookmark this page for easy reference during the exercises.

Schema Generation

sz_schema_generator.py

Analyzes source data and generates detailed schema documentation.

Basic usage:

python3 workshop/senzing/tools/sz_schema_generator.py <input_file> -o <output_schema>.md

For CSV files:

python3 workshop/senzing/tools/sz_schema_generator.py data.csv -o schema.md

For JSON files:

python3 workshop/senzing/tools/sz_schema_generator.py data.jsonl -o schema.md

Output: Markdown file with field analysis, types, sample values, and population rates

Mapping & Validation

Senzing Mapping Assistant (via Amazon Q)

Load the mapping assistant prompt into Amazon Q Developer:

Prompt location:

workshop/senzing/prompts/senzing_mapping_assistant.md

Reference files to load (in Stage 1: INIT):

  • workshop/senzing/reference/senzing_entity_specification.md
  • workshop/senzing/reference/senzing_mapping_examples.md
  • workshop/senzing/reference/identifier_crosswalk.json
  • workshop/senzing/reference/usage_type_crosswalk.json

Process: Follow the 5 stages (INIT → INVENTORY → PLANNING → MAPPING → OUTPUTS)

lint_senzing_json.py

Validates that JSON output is syntactically correct and has required fields.

Usage:

python3 workshop/senzing/reference/lint_senzing_json.py <output>.jsonl

What it checks:

  • Valid JSON syntax
  • Required fields present (DATA_SOURCE, RECORD_ID)
  • Line-by-line validation with error reporting

Output: Error messages if issues found, success message if clean

sz_json_analyzer.py

Analyzes mapped Senzing JSON to show which attributes are recognized.

Usage:

python3 workshop/senzing/tools/sz_json_analyzer.py <output>.jsonl

Color-coded output:

  • GREEN: Recognized Senzing features (used for matching)
  • YELLOW: Payload attributes (stored but not matched)
  • RED: Errors (unregistered data sources, invalid attributes)
  • ORANGE: Warnings (sparsely populated fields)

Useful flags:

# Show detailed statistics
python3 workshop/senzing/tools/sz_json_analyzer.py <output>.jsonl --stats

# Filter to specific data source
python3 workshop/senzing/tools/sz_json_analyzer.py <output>.jsonl --datasource CUSTOMERS

Data Loading

sz_configtool

Manages Senzing configuration, including data source registration.

Add a data source:

sz_configtool add-datasource --datasource <DATA_SOURCE_NAME>

List data sources:

sz_configtool list-datasources

View full configuration:

sz_configtool get-config

Example:

# Register CUSTOMERS data source
sz_configtool add-datasource --datasource CUSTOMERS

Data source names must be uppercase and should match what you specified in your mapper.

sz_file_loader

Loads mapped JSON data into Senzing for entity resolution.

Basic usage:

sz_file_loader --file <input>.jsonl --datasource <DATA_SOURCE_NAME>

Example:

sz_file_loader --file customers_senzing.jsonl --datasource CUSTOMERS

What happens:

  • Reads each JSON record
  • Processes through Senzing engine
  • Resolves entities (finds matches)
  • Stores in Senzing database

Progress: Shows records processed and any errors encountered

Snapshot & Analysis

sz_snapshot

Generates entity resolution analysis files from the Senzing database.

Basic usage:

sz_snapshot --outputDir <snapshot_directory> --datasources <DATA_SOURCE_NAME>

Example:

sz_snapshot --outputDir snapshot_customers --datasources CUSTOMERS

Multiple data sources:

sz_snapshot --outputDir snapshot_all --datasources CUSTOMERS,WATCHLIST

What it creates:

  • Entity statistics
  • Match analysis
  • Relationship graphs
  • Data quality reports
  • Feature usage statistics

Output: Directory with multiple analysis files (.csv, .json, .md)

sz_explorer

Interactive tool for browsing snapshot analysis files.

Basic usage:

sz_explorer <snapshot_directory>/

Example:

sz_explorer snapshot_customers/

What you can do:

  • Browse entities and their constituent records
  • View match details (which features caused matches)
  • Explore relationships
  • Analyze data quality
  • Export findings

Navigation: Interactive menu-driven interface

Senzing MCP Server (via Amazon Q)

Query Senzing directly through natural language using the MCP server integration.

Available through: Amazon Q Developer chat interface

Example queries:

What data sources are configured in Senzing?

Get entity statistics

Search for entities with the name "Smith"

Get entity by ID 1

Explain why records with IDs 1001 and 1002 matched

Show me all entities from the CUSTOMERS data source

What the MCP provides:

  • Entity search by attributes
  • Entity lookup by ID
  • Statistics and configuration info
  • Match explanations (why entities resolved together)
  • Relationship exploration

File Viewing

View CSV files

head -20 data.csv
cat data.csv | less

View JSON files (pretty-printed)

head -3 data.jsonl | python3 -m json.tool
cat data.jsonl | python3 -m json.tool | less

Count records

# CSV (subtract 1 for header)
wc -l data.csv

# JSON Lines
wc -l data.jsonl

Search for specific values

# CSV
grep "Smith" data.csv

# JSON
grep "Smith" data.jsonl

Common Workflows

Complete mapping validation workflow:

# 1. Generate schema
python3 workshop/senzing/tools/sz_schema_generator.py input.csv -o schema.md

# 2. Use Mapping Assistant in Amazon Q (interactive)

# 3. Run the generated mapper
python3 mapper.py input.csv -o output.jsonl

# 4. Validate output
python3 workshop/senzing/reference/lint_senzing_json.py output.jsonl
python3 workshop/senzing/tools/sz_json_analyzer.py output.jsonl

Complete load and analysis workflow:

# 1. Register data source
sz_configtool add-datasource --datasource MYDATASOURCE

# 2. Load data
sz_file_loader --file output.jsonl --datasource MYDATASOURCE

# 3. Take snapshot
sz_snapshot --outputDir snapshot_mydatasource --datasources MYDATASOURCE

# 4. Explore results
sz_explorer snapshot_mydatasource/

Troubleshooting

“Command not found” errors

Make sure you’re in the correct directory:

cd /home/ubuntu/workshop  # Or your workshop root
pwd  # Verify location

Python module import errors

Verify Python path is set:

echo $PYTHONPATH
# Should include: /opt/senzing/sdk/python

Senzing engine errors

Check environment variables:

echo $SENZING_ENGINE_CONFIGURATION_JSON
echo $LD_LIBRARY_PATH

Data source not registered

List registered data sources:

sz_configtool list-datasources

If missing, register it:

sz_configtool add-datasource --datasource <DATA_SOURCE_NAME>
