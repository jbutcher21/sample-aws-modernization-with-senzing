AI-Assisted Solution

How AI Transforms the Mapping Process

Instead of spending days manually mapping data, you can use AI assistance to complete the work in under an hour - with better results and complete documentation.

The Senzing Mapping Assistant

This workshop teaches a structured, repeatable workflow called the Senzing Mapping Assistant. Rather than just asking AI to “write me a mapper,” you follow a 5-stage process that ensures correct, validated mappings every time.

Think of it as a guided conversation with AI where you work together to:

  1. Understand your source data
  2. Plan your approach
  3. Make informed mapping decisions
  4. Generate production-ready code
  5. Validate everything works

This isn’t just code generation - it’s AI-assisted decision-making with YOU in control. The AI guides you through Senzing specifications and best practices, but YOU make the final decisions based on your understanding of the data.

The 5-Stage Workflow

STAGE 1: INIT - Load Reference Materials

Load the Senzing specifications and reference materials into AI context.

What you do:

  • Load the 2,100+ line Senzing entity specification
  • Load mapping examples and crosswalk files
  • Verify tools are available

What the AI gains:

  • Complete knowledge of Senzing features and attributes
  • Real-world mapping examples
  • Best practices for common scenarios

Time: 2-3 minutes

STAGE 2: INVENTORY - Analyze Source Data

The AI analyzes your source data schema to understand what fields you’re working with.

What you do:

  • Run sz_schema_generator.py on your source data
  • Provide the generated schema to AI

What the AI learns:

  • All field names in your source data
  • Data types and sample values
  • Population rates (how often fields are filled)
  • Potential data quality issues

Time: 5-10 minutes

STAGE 3: PLANNING - Identify Entities and Sources

Work with AI to establish your mapping strategy.

What you decide together:

  • What DATA_SOURCE value to use (e.g., “CUSTOMERS”)
  • Are there multiple entity types? (PERSON vs ORGANIZATION)
  • Any conditional logic needed?
  • Special handling requirements?

Key questions AI will ask:

  • “What should we use for the DATA_SOURCE code?”
  • “Does your data contain organizations, persons, or both?”
  • “Are there any conditional fields that determine entity type?”

Time: 5-10 minutes

STAGE 4: MAPPING - Field-by-Field Decisions

This is the core mapping work. For each source field, you and the AI determine the appropriate Senzing mapping.

For each field, decide:

  • Feature (used for entity matching)?
  • Payload (stored but not matched)?
  • Ignore (not needed)?

If it’s a Feature:

  • Which Senzing attribute? (e.g., NAME_FIRST, SSN_NUMBER, ADDR_CITY)
  • Any data transformations needed? (date formats, name parsing, etc.)

What happens:

  • AI suggests mappings based on field names and sample values
  • You confirm or override the suggestions
  • Handle edge cases (like dynamic identifier types)
  • Make transformation decisions

Time: 20-30 minutes (the main work)

STAGE 5: OUTPUTS - Generate Code and Documentation

The AI generates all production deliverables based on your mapping decisions.

Generated outputs:

  1. Mapping specification (.md file) - Complete documentation of all decisions
  2. Python mapper (.py file) - Production-ready executable code
  3. Sample output (.jsonl file) - Example mapped records
  4. README - Usage instructions

What you do:

  • Review the generated mapper code
  • Run the mapper on your source data
  • Validate output with linting and analysis tools

Time: 15-20 minutes

Why This Workflow Works

Structured Decision-Making

Each stage has a clear purpose and deliverable. You never wonder “what do I do next?”

Validation Gates

You verify correctness at each stage before proceeding. Catch issues early.

Iterative Refinement

Easy to go back and adjust decisions if you discover issues during validation.

Documentation Trail

Complete record of WHY each mapping decision was made, not just WHAT was mapped.

Production Ready

Output includes working code, comprehensive documentation, and validated data.

Traditional vs AI-Assisted Comparison

Aspect Traditional Manual Approach AI-Assisted Workflow
Time 3-6 days 1 hour
Specification Reading You read 2,100+ lines AI has full context
Mapping Decisions Manual research per field AI suggests, you decide
Code Writing Write from scratch Generated automatically
Documentation Created afterward (maybe) Generated as byproduct
Validation Manual checking Automated tools
Error Risk High (easy to miss specs) Low (AI knows all specs)
Maintainability Hard to understand later Complete decision trail

What Makes This Different from Simple Code Generation?

This isn’t about asking AI to “write a Senzing mapper for my CSV file” and hoping it works.

The Senzing Mapping Assistant provides:

  1. Structured workflow - A repeatable process, not ad-hoc prompting
  2. Human-AI collaboration - You make decisions, AI provides guidance
  3. Complete context - AI has full Senzing specifications loaded
  4. Validation steps - Tools verify correctness at each stage
  5. Production quality - Generated code is maintainable and documented
  6. Knowledge transfer - You learn entity resolution concepts through the process

Key Insight: By following a structured workflow, you get better results than just asking AI to “do it for me.” You maintain control, understand the decisions, and end up with production-ready outputs.

  • Follow all 5 stages with AI guidance
  • Make real mapping decisions for 19 source fields
  • Handle conditional logic (PERSON vs ORGANIZATION)
  • Deal with dynamic identifiers
  • Generate a working mapper
  • Validate output with professional tools

In Exercise 2 (Watchlist Data), you’ll:

  • Apply the same 5-stage workflow to more complex data
  • Handle nested JSON structures (FollowTheMoney format)
  • Map international names and relationships
  • See how the workflow scales to advanced scenarios
