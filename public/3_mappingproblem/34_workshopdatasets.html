Workshop Datasets

Two Datasets, Same Workflow

This workshop includes two datasets that demonstrate the Senzing Mapping Assistant workflow at different complexity levels. Both follow the same 7-step process you learned in the previous sections.

Dataset 1: Customer Data (Exercise 1)

File: workshop/workspace/customers/customers.csv

Format: CSV with 120 records

Challenge Level: ⭐⭐ Intermediate

What’s in This Dataset?

A realistic customer database export containing:

  • 114 PERSON records - Individual customers
  • 6 ORGANIZATION records - Business customers

This mimics real-world scenarios where a single data source contains multiple entity types.

Data Characteristics

Core Fields:

  • Customer ID and name
  • Customer type indicator (Individual or Company)

Demographics (for persons):

  • Gender and date of birth

Contact Information:

  • Street address, city, state, ZIP, country
  • Phone numbers
  • Email addresses

Identifiers:

  • Dynamic identifier system: id_type, id_number, id_country
  • Types: SSN, Driver’s License, Passport, National ID
  • Challenge: Must map conditionally based on id_type value

Payload Fields (not used for matching):

  • Customer registration date
  • Account status
  • Account balance
  • Customer tier (Bronze/Silver/Gold)

Data Quality: Like real-world data, not every field is populated for every record. You’ll see varying completeness across fields.

Why This Dataset Is Good for Learning

✅ Manageable size - 120 records is enough to see patterns without being overwhelming ✅ Mixed entity types - Teaches conditional mapping (PERSON vs ORGANIZATION) ✅ Dynamic identifiers - Real-world challenge of field-driven logic ✅ Clear business context - Easy to understand what the data represents ✅ Good for validation - Small enough to manually verify some mappings

Dataset 2: Watchlist Data (Exercise 2)

File: workshop/workspace/watchlist/ftm.jsonl

Format: JSON Lines (FollowTheMoney format) with 70 entities

Challenge Level: ⭐⭐⭐ Advanced

What’s in This Dataset?

International watchlist records in FollowTheMoney (FTM) format representing sanctioned individuals and organizations.

Entity Breakdown:

  • 33 Person entities - Sanctioned individuals
  • 6 Company entities - Sanctioned organizations
  • 17 Sanction relationships - Connections to sanctioning bodies
  • 8 Ownership relationships - Who owns what
  • 6 Directorship relationships - Leadership connections

Data Characteristics

Structure:

  • Nested JSON with properties as arrays
  • Multiple entity types in single file
  • Relationships between entities
  • International character sets

Challenges:

  • Complex format: FTM uses nested structures, not flat CSV
  • International names: Various scripts and languages
  • Relationships: Must handle entity connections
  • Multiple schemas: Different properties per entity type

Why This Dataset Is Advanced

⚡ Complex structure - Nested JSON vs simple CSV ⚡ Relationship modeling - Entities reference each other ⚡ International data - Non-ASCII characters, various naming conventions ⚡ Real-world use case - Watchlist screening is a common entity resolution application ⚡ Transfer learning - Tests if you can apply the workflow to unfamiliar formats

Pre-Generated Schema Files

To help you understand data structure before mapping, pre-generated schema analyses are available:

Customer Schema

Location: workshop/solutions/customers/customers_schema.md

Contains:

  • Field-by-field analysis
  • Data type detection
  • Population percentages (how often fields are filled)
  • Sample values from the data
  • Cardinality (number of unique values)

Watchlist Schema

Location: workshop/solutions/watchlist/ftm_schema.md

Contains:

  • FTM structure breakdown
  • Entity type distribution
  • Property analysis per entity type
  • Sample entities

These schema files were generated using workshop/senzing/tools/sz_schema_generator.py - a tool you’ll use in both exercises to analyze source data.

Viewing the Data

You can explore the datasets directly in your environment:

View Customer Data

cd workshop/workspace/customers
head -20 customers.csv

View Watchlist Data

cd workshop/workspace/watchlist
head -3 ftm.jsonl | python3 -m json.tool

Review Pre-Generated Schemas

# Customer schema
cat workshop/solutions/customers/customers_schema.md | head -50

# Watchlist schema
cat workshop/solutions/watchlist/ftm_schema.md | head -50

Take 5 minutes now to browse these files. Familiarize yourself with the data structure before you start mapping. Understanding your source data is the foundation of successful mapping.

What You’ll Do With This Data

In Exercise 1 (Customer Data)

You’ll follow the 7-step workflow to:

  1. Examine the CSV structure
  2. Create a schema using the schema generator tool
  3. Map fields to Senzing format using the Mapping Assistant
  4. Validate output with linter and analyzer tools
  5. Load data into Senzing
  6. Take a snapshot for analysis
  7. Analyze entity resolution results

In Exercise 2 (Watchlist Data)

You’ll apply the same 7-step workflow to the more complex watchlist data, demonstrating that the process is repeatable regardless of source format complexity.

Try it yourself first! The solutions are there if you get stuck or want to compare your approach. The learning happens when you make the mapping decisions yourself with AI guidance.
